# Electrolysis Stimulator (Tech README)

## Tech stack
- Frontend: Next.js (React) + TypeScript
- UI components: Tailwind CSS + shadcn/ui (`Card`, `Button`, `Input`, `Select`, `Slider`)
- Icons: `lucide-react`
- Backend: none for this page (client-side computation only)

## Logic (what the page does)
1. User selects a “product/substance” (e.g., Oxygen/Hydrogen) from a dropdown.
2. User enters:
   - `Current` in amperes (I)
   - a time duration as `hours:minutes:seconds` (t)
   - an efficiency factor (0–100%)
3. When the user clicks “Calculate Yield”, the page computes the theoretical electrolysis yield using Faraday’s laws.
4. Results are displayed in four cards: theoretical yield (mass), moles produced, total charge passed, and electrons transferred.

## Technical concepts used
- Faraday’s laws of electrolysis (charge–amount-of-substance relationship)
- Dimensional analysis (implemented as a “train tracks” style explanation in the UI)
- Client-side form state + derived calculations
- Basic input validation (disable the calculate button unless required inputs exist)

## Algorithms (implemented calculation steps)
### 1) Time to seconds
The code converts `hours`, `minutes`, `seconds` into a total duration:
`t = (h * 3600) + (m * 60) + s`

### 2) Apply efficiency to charge
- Faraday’s constant: `F = 96485 C/mol`
- Efficiency factor: `η = efficiency / 100`
- Total charge:
  `Q = I * t * η`

### 3) Electrons to moles
- Moles of electrons transferred:
  `n_e = Q / F`

### 4) Convert to moles of product (valency)
The selected substance has `valency` interpreted as “electrons required per produced ion/molecule unit” (`n` in the code):
`molesProduced = n_e / n`

### 5) Moles to theoretical yield (mass)
`theoreticalYield = molesProduced * molarMass`

### 6) Electrons transferred (count)
The UI also reports:
`electronsTransferred = n_e * 6.022e23`

## Data structures
- `interface Substance`
  - `name`, `formula`, `molarMass`, `valency`, `description`
- `const substances: Substance[]`
  - Hardcoded product list used by the selector
- React state:
  - input state: `current`, `hours`, `minutes`, `seconds`
  - `efficiency: number[]` (slider output; code uses `efficiency[0]`)
  - `results` object containing computed values

## Dataset(s)
### Hardcoded “substance/product” dataset
No external dataset file is used. The dropdown is populated from an in-code array containing:
- Copper (`Cu`)
- Aluminum (`Al`)
- Sodium (`Na`)
- Chlorine (`Cl₂`)
- Oxygen (`O₂`)
- Hydrogen (`H₂`)
