# Interactive Periodic Table (Tech README)

## Tech stack
- Frontend: Next.js (App Router) + React + TypeScript
- UI components: Tailwind CSS + shadcn/ui (`Card`, `Button`, `Badge`)
- Data loading:
  - Client-side CSV fetch (`fetch('/data/PeriodicTableCSV.csv')`)
  - Custom CSV parsing in TypeScript (supports quoted CSV fields)

## Logic (what the page does)
The page `frontend/app/periodic/page.tsx` provides 3 interactive views over all elements:
1. **Periodic Table view**: grid layout (plus separate lanthanide/actinide strips). Clicking an element shows its details.
2. **Elements Property view**: elements grouped into categories (alkali metals, halogens, noble gases, etc.). Selecting a category shows a list of elements and key properties.
3. **Periodic Trends view**: interactive line/point plots of element properties (electronegativity, ionization energy, electron affinity, atomic radius), plus min/max and average statistics.

Data loading path:
1. On mount, `loadElementsFromCSV()` loads the CSV and parses it into an `Element[]`.
2. The UI uses the parsed `Element[]` to build:
   - the periodic grid
   - category-filtered lists
   - trend series arrays

## Technical concepts used
- Client-side data ingestion + parsing
  - Fetching a CSV file as text
  - Parsing with a small state machine to handle:
    - commas inside quotes
    - quoted fields
  - Normalizing CSV columns to the `Element` interface
- Derived state & conditional rendering
  - `viewMode` selects which view is displayed
  - `selectedCategory` / `selectedElement` drive detail rendering
- Simple data-mapping algorithms
  - mapping `period/group` to grid coordinates
  - computing chart points from numeric series (min/max normalization)

## Algorithms (important implemented computations)
### 1) Periodic table grid placement
The page builds:
- `mainGrid`: 7 rows × 18 columns (`Array(7).fill(...).map(...)`)
- `lanthanides` and `actinides` arrays by atomic number ranges:
  - Lanthanides: atomicNumber 57–71
  - Actinides: atomicNumber 89–103

For main-group elements, placement uses:
- `row = element.period - 1`
- `col = element.group - 1`
and then checks bounds before inserting into `mainGrid[row][col]`.

### 2) Trend chart point generation
`buildTrendPoints(values, width, height)`:
- filters out NaN values to compute `min`/`max` and `range`
- assigns each data index `i` an X coordinate:
  - `x = (i / (values.length - 1)) * width`
- assigns Y coordinate with min/max normalization
- missing/NaN values map to a “bottom” Y (`height`)

### 3) Category view filtering
`ElementsPropertyView` uses a fixed category list and filters elements by `element.category`.
For the “nonmetal” family, it treats multiple related categories as equivalent:
- `nonmetal`, `diatomic nonmetal`, `polyatomic nonmetal`

## Data structures
- `Element` interface (`frontend/lib/periodic-table-data.ts`):
  - holds atomicNumber/symbol/name + physical/periodic properties
  - includes optional values (electronegativity, atomicRadius, ionizationEnergy, etc.)
  - stores presentation helpers like `color` and optional image URL/attribution fields
- `elements: Element[]`:
  - full dataset loaded from CSV
- Periodic grid matrices:
  - `mainGrid: (Element | null)[][]`
  - `lanthanides: Element[]`
  - `actinides: Element[]`
- UI state:
  - `viewMode: 'table' | 'property' | 'trends'`
  - `selectedCategory: string | null`
  - `selectedElement: Element | null`

## Dataset(s)
### Element dataset (CSV)
- `frontend/public/data/PeriodicTableCSV.csv`
  - The CSV header includes (at least) these fields:
    - identification & grouping: `number`, `symbol`, `name`, `period`, `group`, `block`
    - physical/periodic properties: `atomic_mass`, `density`, `melt/boil`, `electronegativity_pauling`, `ionization_energies`, `electron_affinity`, etc.
    - visuals & assets: `bohr_model_image`, `bohr_model_3d`, `image.url`, `image.attribution`

### Where the data is used
- `frontend/lib/csv-parser.ts` maps CSV columns into the `Element` objects.
- `frontend/app/periodic/page.tsx` consumes:
  - `element.period`, `element.group`, `element.atomicNumber` for grid placement
  - `element` properties for detail cards
  - numeric properties for trend series computation
